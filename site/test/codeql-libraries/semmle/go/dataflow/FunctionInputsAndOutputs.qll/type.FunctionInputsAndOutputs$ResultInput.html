Class ResultInput

A result position of a function, viewed as an input.

Results are usually outputs rather than inputs, but for taint tracking it can be useful to think of taint propagating backwards from a result of a function to its arguments. For instance, the function bufio.NewWriter returns a writer bw that buffers write operations to an underlying writer w. If tainted data is written to bw, then it makes sense to propagate that taint back to the underlying writer w, which can be modeled by saying that bufio.NewWriter propagates taint from its result to its first argument.
